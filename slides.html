What is Agent Orchestration?

A system where specialized AI agents collaborate — each with its own model, role, and expertise — coordinated by an orchestrator.

🎯

Custom Agents

Define agents in simple Markdown files with specific roles, models, and tools

🔀

Sub-Agents

Agents delegate work to other agents — the orchestrator never writes code itself

⚡

Parallelization

Non-conflicting tasks run simultaneously, dramatically reducing build time

The Architecture

Four specialized agents, three AI providers, one orchestrator

🎭 Orchestrator Claude Sonnet 4.6
📋 Planner GPT-5.4
💻 Coder GPT-5.3-Codex
🎨 Designer Gemini 3.1 Pro

Key principle: The Orchestrator coordinates but never implements. It breaks down requests, delegates to specialists, and manages execution phases.

Agent Definitions are just Markdown

Stored in .github/agents/ — simple, version-controlled, shareable

--- name: Orchestrator description: Sonnet, Codex, Gemini model: Claude Sonnet 4.6 (copilot) tools: ['read/readFile', 'agent', 'memory'] --- You are a project orchestrator. You break down complex requests into tasks and delegate to specialist subagents. You coordinate work but NEVER implement anything yourself.
orchestrator.agent.md planner.agent.md coder.agent.md designer.agent.md

Meet the Agents

Claude Sonnet 4.6

🎭 Orchestrator

Breaks requests into phases. Detects parallelism. Delegates to specialists. Never writes code.

Tools: readFile, agent, memory

GPT-5.4

📋 Planner

Researches codebase. Checks docs via Context7. Outputs WHAT to build, not HOW. Identifies edge cases.

Tools: vscode, execute, read, context7, web

GPT-5.3-Codex

💻 Coder

Implements code following strict principles. Always checks latest docs. Focuses on regenerability.

Tools: vscode, execute, edit, context7, github

Gemini 3.1 Pro

🎨 Designer

Controls all UI/UX decisions. Prioritizes user experience over technical constraints. Owns the design.

Tools: vscode, execute, edit, context7, web

Multi-Model: The GitHub Copilot Edge

Use the best model for each job — not just one model for everything

🏆 GitHub Copilot

  • Claude (Anthropic) for orchestration
  • GPT-5.x (OpenAI) for planning & coding
  • Gemini (Google) for design
  • Mix and match per agent
  • Each agent uses its optimal model

Other Tools

  • Single model provider only
  • Claude Code → Anthropic only
  • Can't mix providers per agent
  • Same model for all tasks
  • No model specialization

This is a unique differentiator — GitHub Copilot lets you assign different AI providers to different agents, picking the best model for each role.

How It Works

The orchestrator follows a structured 4-step execution model

📋 Get Plan Call Planner agent
→
🧩 Parse Phases Detect parallelism
→
⚡ Execute Spawn sub-agents
→
✅ Verify Report results

Parallelization rules: Tasks touching different files or different domains run in parallel. Tasks with data dependencies or shared files run sequentially to prevent conflicts.

Parallel Execution in Action

Non-conflicting tasks run simultaneously across agents

Phase 1 — Plan

📋 Planner: Create strategy

Sequential — plan needed first

Phase 2 — Build ⚡ Parallel

💻 Coder: Game engine & logic
🎨 Designer: UI/UX & styling

Different files → run simultaneously

Phase 3 — Integrate

💻 Coder: Wire everything up

Sequential — depends on Phase 2

Key Takeaways

Insight

Model Selection Matters Most for Orchestrator & Coder

These roles had the largest impact on output quality. Planner and Designer showed less sensitivity to model tier.

Insight

Speed vs. Quality is a Real Tradeoff

"Fast" model variants reduce time but cost you features and correctness. Choose wisely per use case.

Insight

Multi-Model = Pick the Best Tool

Claude orchestrates well, Codex codes clean, Gemini designs beautifully. Each model has strengths — use them.

Insight

Agent Definitions are Simple & Portable

Just Markdown files in your repo. Version-controlled, shareable, customizable. No complex infrastructure needed.
